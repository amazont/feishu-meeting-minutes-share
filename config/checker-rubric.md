# Checker 评分标准（外置 rubric）

> 本文件是生产 checker 的**唯一评分标准来源**（workflow 通过 args.rubricFile 注入，checker agent 逐条对照打分）。
> 进化系统调整评分标准时只改本文件；改动经离线回归验证后随 release 快照晋升生效。
> 版本追溯：以本文件的 git 提交历史为准。

C1 覆盖度:七个必备二级标题(一、一句话结论/二、背景与目的/三、分议题讨论/四、形成的决策/五、Action Items/六、需要继续确认的问题/七、相关链接)齐全且非空;「八、备注」可选,存在则须非空。其中「一、一句话结论」段以一张「核心结论看板」画板(`<whiteboard type="svg">`)承载,该画板须存在且其 `<text>` 含实质结论文字(非空壳)。

C2 正确性:各段内容忠实于源妙记(notes/逐字稿),不臆造结论、决策、待办或开放问题;「一句话结论」看板里的核心结论高亮带 + 各卡片要点,必须是"这个会定了什么方向/得出什么判断"的提炼(从草稿内联 SVG 的 `<text>` 读取核对),不是议程复述。

C3 可验证性:相关链接指向有效 minute URL;首行标题格式为「# YYYY-MM-DD 概括标题 会议纪要」,中间的概括标题 10-20 字且不是「某某的视频会议」这类默认名。

C4 一致性/排版:Lark-flavored Markdown —— 首行 `# 标题`、一个 <callout> 会议信息块(每行单个 <br/> 分隔、不用 <br/><br/>)、块之间有空行;「一、一句话结论」段正文是且仅是一个合法 `<whiteboard type="svg">` 画板(自包含 `<svg viewBox>`、文字用 `<text>`、含结论高亮带 + 2–4 张要点卡片、不含 `<radialGradient>`/`<filter>`/`<clipPath>`/`<mask>` 等画板不支持特性、CJK 文本不溢出),飞书能正确渲染成画板;「三、分议题讨论」按议题/发言人用「### 三级标题 + 叙事段落 + 要点列表」组织,不得按时间戳机械切片、不得把整段正文挤进单个列表项;「四、形成的决策」是有序编号列表(1. 2. 3.),每条一句完整判断,不用表格;「五、Action Items」是三列表格 | 负责人 | 待办事项 | 时间节点 |,单元格不含 <br/> 与 @用户/<cite> 提及标记,负责人写纯姓名文本;飞书能正确渲染。
